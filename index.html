Programming languages I like to use

  • Python - Pretty much my go-to language. I like its simplicity and how it's written in a way where the syntax just makes a lot of sense. The more I've used Python, the greater appreciation I have for how powerful its built-in functions can be and how efficient they can make a piece of code.

  • Perl - I kind of think of Perl like "Bash++". Really good for interfacing with the system and for more complex system programming scripts. Used a lot at one of my previous jobs, but haven't really been motivated to do much with it since. Python kind of replaces any more complex tooling or web design I would do with this. Regardless, I still have an affinity towards Perl and think it can be useful in the right situations. When coding in Perl, I try to make things as readable and understandable to an external user as possible.

  • C/C++ - I learned in C++ and taught myself C through the book C Programming: A Modern Approach by K.N. King. I don't have much of a need to use these languages frequently, but I like to play around with them from time to time. I like to have them in my back pocket as lower-ish level languages I'm familiar with. I think writing in them gives me a better understanding of how more complex algorithms and data structures are designed.

  • Bash - Its syntax is ugly, but I'm comfortable enough with it to write some decent utilities and system scripts.

Programming languages I'm learning how to use better

  • C++ - I know this one decently well, but re-learning it has helped me learn more about programming in general. There's not really an end in sight for how comfortable you can get with a language like this, so it's something I'm pushing myself to learn better and I think it makes me a better programmer in any language.

  • Rust - I honestly don't know much about Rust, but it has a cool name, and sounds like it has the potential to be a more modern replacement for C/C++. So far, have worked through some Project Euler problems with it.
